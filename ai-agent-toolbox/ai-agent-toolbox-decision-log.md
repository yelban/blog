## 2026-01-28 專家諮詢紀錄

### 諮詢背景

**原始問題**：
- 用 Agent Skills 幫中小企業做 AI 轉型服務，這樣能賺錢嗎？
- 現有的 LINE Bot / Telegram Bot / 新聞整理實作，怎麼變成可複製的產品線？
- 用訂閱制（而非 API 計費）來降低成本，這個模式可行嗎？

**諮詢方法**：Best Minds 模擬器思維
> 不問「你怎麼看」，而是問「這個問題，世界上誰最懂？TA 會怎麼說？」

---

### 第一輪諮詢：產品化與商業模式

#### 召喚的專家

| 專家 | 背景 | 選擇理由 |
|------|------|----------|
| **Jason Lemkin** | SaaStr 創辦人 | SMB SaaS 定價與續約策略 |
| **Geoffrey Moore** | 《跨越鴻溝》作者 | 技術產品進入主流市場 |
| **Thomas Davenport** | MIT/哈佛教授 | 企業 AI 應用落地 |
| **Clayton Christensen** | 破壞式創新理論 | 低端切入策略 |

#### 專家意見摘要

**Jason Lemkin**：
> *「你的報告把 Skills 講得很清楚，但犯了 SaaS 新手最常見的錯：試圖賣技術能力，而不是賣結果。」*

- 不要賣「技能包」，要賣「解決的問題」
- 三層產品架構太複雜，應簡化成 3 個價格方案
- SMB 續約關鍵是 switching cost 和 habit formation

**Geoffrey Moore**：
> *「你現在在鴻溝左邊——只有技客和早期採用者會用。問題是：誰是你的『保齡球道第一瓶』？」*

- 不要打「中小企業」這個抽象市場
- 建議第一瓶：10-50 人專業服務公司（會計師、律師、顧問）
- 完整產品 = 技能 + 執行環境 + 入口 + 範本 + 顧問 + 案例 + 推薦

**Thomas Davenport**：
> *「你的報告有個重要的缺口：沒有談『人』。企業 AI 落地失敗，80% 不是技術問題，是組織問題。」*

- 採用阻力：中階主管怕被取代、員工不信任 AI
- 需要內建：可解釋性、人在迴圈、漸進式導入
- 建議加入「導入路徑與變革管理」章節

**Clayton Christensen**：
> *「破壞式創新的核心洞察是：主流客戶『過度被服務』了。你不需要跟大廠比功能。」*

- Anthropic/OpenAI 不會來搶你的生意
- 技術門檻是護城河，不是瓶頸
- 競爭對手不是大廠，是「客戶決定繼續用人工」

#### 決策與修訂

| 原本 | 修訂 | 依據 |
|------|------|------|
| 打「中小企業」市場 | 鎖定「10-50 人專業服務公司」 | Geoffrey Moore |
| 三層產品架構 | 三個定價方案 | Jason Lemkin |
| 6 個範例平均著墨 | 集中 1-2 個打深 | Geoffrey Moore |
| 沒有導入路徑 | 三階段導入法 | Thomas Davenport |
| 技術細節過多 | 砍掉客戶不需要知道的 | Jason Lemkin |

### 第二輪諮詢：成本秘訣的挑戰

#### 召喚的專家

| 專家 | 背景 | 選擇理由 |
|------|------|----------|
| **Chamath Palihapitiya** | Social Capital 創辦人 | 套利模式投資 |
| **Patrick McKenzie (patio11)** | Stripe、SaaS 定價專家 | 成本結構變競爭優勢 |
| **Tomasz Tunguz** | Theory Ventures | SaaS 單位經濟學 |

#### 專家意見摘要

**Chamath Palihapitiya**：
> *「你發現了一個套利機會：訂閱制 AI 服務的定價沒有考慮到 B2B 轉售。這不會永遠存在。」*

- 這是時間有限的視窗（12-24 個月）
- 可能的封堵：條款修改、頻率限制、官方 B2B 方案
- 策略：快速獲客 → 建立黏性 → 轉換護城河
- 套利利潤不是護城河，客戶關係才是

**Patrick McKenzie (patio11)**：
> *「你以為你在賣『便宜的 AI 服務』，但你真正在賣的是『不需要理解 AI 就能用 AI』的能力。」*

- 定價不應基於成本，應基於客戶願意付多少解決問題
- NT$ 8,000 = 「不被認真對待」的價格
- 成本優勢的正確用法：提供更好服務，不是降價
- Charge more.

**Tomasz Tunguz**：
> *「你的毛利分析漏掉了最重要的成本：你的時間。」*

- 90% 毛利率只在時間免費時成立
- 真正的毛利率（含時間成本）：15-55%
- 入門方案應全自助，專業以上才有人力服務

#### 決策與修訂

| 項目 | 原本 | 修訂 | 依據 |
|------|------|------|------|
| 入門定價 | NT$ 8,000 | NT$ 15,000 | patio11 |
| 專業定價 | NT$ 25,000 | NT$ 45,000 + 導入費 | patio11 + Tomasz |
| 旗艦定價 | NT$ 60,000 | NT$ 100,000+ | patio11 |
| 毛利率認知 | 90% | 15-55%（含時間） | Tomasz |
| 入門方案服務 | 含人力 | 全自助 | Tomasz |
| 成本優勢用途 | 降價 | 提供更好服務 | patio11 |
| 視窗認知 | 無 | 12-24 個月有限視窗 | Chamath |
| Plan B | 無 | API 計費備案定價 | Chamath |

## 關鍵決策總結

### 市場定位

```
原本：中小企業（模糊）
   ↓
修訂：10-50 人專業服務公司（會計師、律師、顧問）
理由：工作本質契合 Skills、決策鏈短、同業會互相推薦
```

### 定價策略

```
原本：基於成本 + 客戶能接受
   ↓
修訂：基於客戶願意為解決問題付多少
理由：便宜不是優勢，便宜是跟自己競爭
```

### 服務分層

```
原本：三層都有人力服務
   ↓
修訂：入門全自助、專業含人力、旗艦客製化
理由：時間成本是真實成本，要設計進服務分層
```

### 成本優勢定位

```
原本：用來降價搶客
   ↓
修訂：用來提供競爭對手做不到的服務等級
理由：成本優勢是建立護城河的資本，不是護城河本身
```

### 風險認知

```
原本：訂閱套利是長期優勢
   ↓
修訂：12-24 個月有限視窗，需準備 Plan B
理由：供應商遲早會封堵
```
