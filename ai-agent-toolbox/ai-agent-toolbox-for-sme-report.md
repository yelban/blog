## 0. 執行摘要

### 這樣能賺錢嗎？

**能。** 但不是賣「Skills」，而是賣「省下的時間與降低的風險」。

### 三個核心結論

| # | 結論 | 行動 |
|---|------|------|
| 1 | 客戶不買技術，買結果 | 定價按「省多少」而非「功能多少」 |
| 2 | 先打贏一個小市場 | 鎖定 10-50 人專業服務公司 |
| 3 | 最大競爭對手是「維持現狀」 | 導入路徑比功能重要 |

## A. Agent Skills 是什麼（給老闆看的版本）

### 一句話定義

**Skills 是把「公司的作業知識」變成「AI 可以執行的指令包」。**

不是聊天機器人。是你把「報價怎麼做」「請款流程是什麼」「客戶常問問題怎麼回」寫成 AI 看得懂的格式，然後 AI 就能照著做。

### 跟以前有什麼不同

| 以前 | 現在（Skills） |
|------|----------------|
| 寫程式自動化 → 要工程師 | 用自然語言描述流程 → 不用工程師 |
| 流程圖 + 分支 → 改一個要改整串 | 模組化技能 → 換一塊不影響其他 |
| 操作手冊是手冊，程式是程式 | 技能本身就是可執行的操作手冊 |
| 例外靠人處理 | 例外交給 AI 判斷，確定的交給程式 |

![Skills 概念：從混亂的知識到模組化技能](illustrations/01-framework-skills-concept.png)

### 企業最在意的四件事，Skills 怎麼解決

| 企業在意 | Skills 怎麼做 |
|----------|---------------|
| **交接困難** | 技能檔案本身就是文件，新人讀完就能用 |
| **改流程很痛** | 改一個技能不影響其他，可以版本控管 |
| **怕 AI 亂搞** | 可設定「只能讀不能改」、「要人確認才執行」 |
| **換供應商被綁死** | 開放標準，Anthropic 和 OpenAI 都支援 |

## B. 目標市場：先打一個，打到贏

### 為什麼不能打「中小企業」

「中小企業」包含：
- 賣雞排的
- 做模具的
- 會計師事務所
- 網紅經紀公司

他們的痛完全不同。你不可能用同一套產品打所有人。

### 建議的「保齡球第一瓶」：10-50 人專業服務公司

| 條件 | 為什麼選他們 |
|------|--------------|
| **工作本質** | 把知識變成文件（報告、合約、建議書）→ Skills 完美契合 |
| **決策鏈** | 老闆一個人決定，不用跑採購流程 |
| **同業交流** | 會計師認識會計師，律師認識律師，一家成功會互相推薦 |
| **痛點明確** | 助理貴、流動率高、交接痛苦 |

![保齡球策略：先打一個，打到贏](illustrations/02-infographic-bowling-strategy.png)

**具體對象**：
- 會計師事務所
- 法律事務所
- 管理顧問公司
- 專利/商標事務所
- 建築師/室內設計事務所

### 這群人的具體痛點

| 痛點 | 頻率 | 你能解決的程度 |
|------|------|----------------|
| 回覆客戶常見問題 | 每天 | 90%（限定文件問答） |
| 整理會議紀錄與待辦 | 每天 | 80%（摘要+行動項目） |
| 製作報價單/提案 | 每週 | 70%（套模板+風險標記） |
| 新人訓練/交接 | 每季 | 60%（知識庫+問答） |
| 合約/文件審閱 | 每週 | 50%（標記重點+比對範本） |

## C. 產品設計：簡單到老闆秒懂

### 三個方案（不是三層架構）

| 方案 | 月費 | 包含 | 適合 |
|------|------|------|------|
| **入門** | NT$ 8,000 | LINE 問答 + 1 個文件庫（50 份內）+ 基本紀錄 | 想試試看 |
| **專業** | NT$ 25,000 | LINE + Web + 3 個流程技能 + 客製模板 + 專屬支援 | 認真要用 |
| **旗艦** | NT$ 60,000 | 專業版 + 系統整合 + 進階稽核 + 優先更新 | 要全面導入 |

### 為什麼這樣定價

**Jason Lemkin 的 SMB 定價法則**：
- 月費要低到「老闆不用問老婆」（< NT$ 10,000）才能快速成交
- 但要有升級空間讓 LTV（客戶終身價值）成長
- 年繳打折是標配，讓現金流更好

**建議**：
- 入門方案年繳 = 10 個月（NT$ 80,000/年）
- 專業方案年繳 = 10 個月（NT$ 250,000/年）

### 你賣的不是功能，是結果

| 不要說 | 要說 |
|--------|------|
| 「技能包」 | 「每月省 40 小時行政工作」 |
| 「受控執行環境」 | 「不用碰任何技術，LINE 上就能用」 |
| 「Skills 可組合」 | 「流程變了隨時能改，不用重新開發」 |
| 「最小權限控管」 | 「AI 只能看不能改，重要的事一定先問你」 |

## D. 導入路徑：沒有這個就不會落地

### Thomas Davenport 的警告

> *「企業 AI 落地失敗，80% 不是技術問題，是組織問題。」*

### 三階段導入法

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1：旁觀期（週 1-2）                                        │
│  ───────────────────────                                        │
│  • AI 產出給人看，人還是自己做                                      │
│  • 目的：建立信任、找出 AI 會錯的地方                                │
│  • 指標：員工願意看 AI 建議的比例                                   │
├─────────────────────────────────────────────────────────────────┤
│  Phase 2：輔助期（週 3-4）                                        │
│  ───────────────────────                                        │
│  • AI 產出草稿，人修改後執行                                        │
│  • 目的：讓人習慣、累積修改樣本優化 AI                               │
│  • 指標：草稿採用率、修改幅度                                       │
├─────────────────────────────────────────────────────────────────┤
│  Phase 3：接管期（月 2+）                                         │
│  ───────────────────────                                        │
│  • AI 執行，人抽查                                                │
│  • 目的：真正省時間                                               │
│  • 指標：每週省下的工時                                            │
└─────────────────────────────────────────────────────────────────┘
```

![三階段導入法：從旁觀到接管](illustrations/03-flowchart-adoption-phases.png)

### 每個階段你要做的事

| 階段 | 你的交付 | 時數估計 |
|------|----------|----------|
| 導入前 | 流程盤點、文件收集、環境設定 | 8 hr |
| Phase 1 | 開通帳號、教育訓練、每日檢視 | 4 hr |
| Phase 2 | 優化調整、例外處理設計 | 4 hr |
| Phase 3 | 效益報告、下一流程規劃 | 2 hr |

**總計**：約 18-20 小時 = 這就是你「一次性導入費」的成本基礎

### 採用阻力與對策

| 阻力來源 | 他們怕什麼 | 你的對策 |
|----------|------------|----------|
| 老闆 | 花錢沒效果 | ROI 試算表 + 第一個月無效退費 |
| 中階主管 | 被取代 | 定位成「讓你專注高價值工作」 |
| 員工 | 不信任 AI | Phase 1 讓他們「看著 AI 做」 |
| IT/資安 | 資料外洩 | 權限控管文件 + 資料處理流程圖 |

## E. ROI 計算：讓老闆秒懂值不值得

### 範例：20 人會計師事務所

**現況成本**：

| 項目 | 每月工時 | 時薪估計 | 月成本 |
|------|----------|----------|--------|
| 回覆客戶基本問題 | 40 hr | NT$ 400 | NT$ 16,000 |
| 整理會議紀錄 | 20 hr | NT$ 400 | NT$ 8,000 |
| 製作報價單初稿 | 15 hr | NT$ 500 | NT$ 7,500 |
| 新人交接說明 | 10 hr | NT$ 500 | NT$ 5,000 |
| **合計** | **85 hr** | | **NT$ 36,500** |

**導入後**：

| 項目 | 節省比例 | 節省工時 | 節省金額 |
|------|----------|----------|----------|
| 回覆客戶基本問題 | 80% | 32 hr | NT$ 12,800 |
| 整理會議紀錄 | 70% | 14 hr | NT$ 5,600 |
| 製作報價單初稿 | 60% | 9 hr | NT$ 4,500 |
| 新人交接說明 | 50% | 5 hr | NT$ 2,500 |
| **合計** | | **60 hr** | **NT$ 25,400** |

**ROI**：
- 專業方案月費：NT$ 25,000
- 每月節省：NT$ 25,400
- **第一個月就回本，之後每月淨省 NT$ 400 + 60 小時可做更高價值的事**

### 給老闆的一句話

> 「你請一個助理月薪 3-4 萬，還要訓練、會請假、會離職。這個 AI 助理月費 2.5 萬，24 小時待命，永遠不會忘記你教過的事。」

## F. 你現有實作的產品化路徑

### 1. LINE Bot → 「專業服務公司內規問答機器人」

**現有能力**：
- 上傳 PDF 限定範圍問答
- 對話呼叫 Skills
- Gemini 圖片生成

**產品化**：

| 功能 | 包裝成 | 賣點 |
|------|--------|------|
| PDF 問答 | 「內規/合約/手冊即時查詢」 | 新人不用問前輩 |
| 對話呼叫 Skills | 「用說的完成工作」 | 不用開電腦 |
| 圖片生成 | 「簡報配圖/社群貼文」 | 不用等設計 |

**第一個產品**：
- 名稱：「所內小助理」（針對事務所）
- 功能：上傳所內規範/範本 → LINE 問答 → 找到相關規定/範本
- 定價：NT$ 8,000/月（50 份文件內）

### 2. Telegram Bot + macOS → 「老闆私人助理」

**現有能力**：
- 對話 → 加到 Apple 行事曆
- 對話 → 寄信
- 對話 → 整理檔案

**產品化**：

| 功能 | 包裝成 | 賣點 |
|------|--------|------|
| 行事曆 | 「用說的排行程」 | 開車/走路時也能排 |
| 寄信 | 「用說的寫信」 | 口述大意，AI 潤稿 |
| 檔案整理 | 「用說的找檔案」 | 不用記放哪裡 |

**定位**：這是「旗艦方案」的加值功能，不單獨賣。

### 3. 新聞抓取 + 評分 + TTS → 「產業情蒐晨報」

**現有能力**：
- 抓取指定來源
- Skills 評分篩選
- 網頁呈現 + 語音播報

**產品化**：

| 功能 | 包裝成 | 賣點 |
|------|--------|------|
| 抓取+評分 | 「只看重要的」 | 每天 5 分鐘掌握產業動態 |
| 語音播報 | 「通勤時用聽的」 | 眼睛不用盯螢幕 |
| 定期更新 | 「每天自動送到 LINE」 | 不用自己去找 |

**定位**：專業方案的加值模組，NT$ 5,000/月加購。

## G. 競爭分析：你的敵人不是大廠

### Clayton Christensen 的洞察

> *「你的競爭對手不是 Anthropic，是『客戶決定繼續用 Excel 和 LINE 群組人工處理』。」*

### 競爭格局

| 競爭者 | 他們的定位 | 為什麼不用怕 |
|--------|------------|--------------|
| Anthropic/OpenAI | 賣 API 給大企業 | 不會來幫 30 人事務所設定環境 |
| 大型 SI（系統整合商） | 百萬級專案 | 中小企業買不起 |
| 其他 AI 新創 | 多半做垂直 SaaS | 你做的是水平能力 |
| **維持現狀** | 繼續用人工 | **這才是真正對手** |

### 你的護城河

| 護城河 | 說明 |
|--------|------|
| **技術門檻** | 客戶不會自己搞 Shell/npm/權限 |
| **使用習慣** | 一旦習慣用 LINE 問 AI，就很難換 |
| **累積的客製** | 客戶的模板、規則、例外處理都在你這 |
| **信任關係** | 中小企業買人不買產品 |

## H. 風險與護欄：把這變成銷售優勢

### 企業最怕的三件事

| 怕什麼 | 你怎麼解決 | 怎麼講 |
|--------|------------|--------|
| AI 亂改東西 | 預設只能讀，改東西要人確認 | 「AI 只能建議，按下確認才會動」 |
| 資料外洩 | 權限分級 + 紀錄誰看了什麼 | 「誰問了什麼、AI 回了什麼，全部有紀錄」 |
| AI 出錯怎麼辦 | 輸出必附引用來源 | 「每個回答都告訴你根據哪份文件」 |

### 標準話術

> 「我們的 AI 有三個保險：
> 1. **只讀不改**——除非你按確認
> 2. **全程紀錄**——誰問什麼、AI 答什麼
> 3. **必附出處**——每個答案都說根據哪裡」

# 所內小助理

## 讓 AI 回答同仁的日常問題

**痛點**：
- 新人一直問同樣的問題
- 資深同仁被打斷工作
- 內規/範本散落各處找不到

**解法**：
上傳所內文件 → 同仁用 LINE 問 → AI 秒回答案並附出處

**效果**：
- 每月省 30+ 小時重複回答
- 新人即時得到答案
- 知識不會隨人離職

**費用**：
- 月費 NT$ 8,000（50 份文件內）
- 首月無效全額退費

**下一步**：
掃 QR code 預約 30 分鐘線上展示
